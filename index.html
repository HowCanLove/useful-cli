为什么用现代 CLI 工具？

几十年来，bash 用户被 ls / find / grep / cat 这些 Unix 经典工具的默认行为折磨：彩色要手动加 --color=auto、find 语法反人类、grep 不识别 .gitignore、cat 看大文件没分页。这些工具诞生于 70–80 年代，设计早已跟不上现代开发节奏。

bat、ripgrep、fd、eza 等现代 CLI 神器用 Rust / Go 重写经典工具，秉承同样的 Unix 哲学（一个工具做好一件事、可组合、文本流），但默认值完全为 2020 年代设计：彩色输出、人类可读单位、自动尊重 .gitignore、并行执行、毫秒级启动。lazygit、k9s、btop 等 TUI 工具进一步把命令行变成「全屏交互界面」，让 Git 操作、Kubernetes 集群、系统监控不用再背几十个命令。starship、zellij、atuin 这一类则改造 Shell 本身——更聪明的 prompt、更顺手的多窗格、跨机器同步的命令历史。

本目录精选数十款经实战验证的现代终端工具，按 CLI 命令 / TUI 全屏 / Shell 美化 三类分组。全部开源免费，大部分支持 Homebrew / Cargo / apt / Scoop 一键安装，Linux、macOS、Windows（含 WSL）通用。装上之后，你会发现自己再也回不去 GNU coreutils。

所有工具完整列表

共 33 款现代 CLI / TUI 工具，覆盖系统监控、文件搜索、文本数据处理、网络调试、Docker / Kubernetes、Git、AI 终端、Shell 美化等场景。

  1. bat

    cat 的现代版：语法高亮 + 行号 + Git 修改指示器 + 自动分页。看代码 / 配置文件再也不会眼瞎。Rust 写的，启动飞快。

    https://github.com/sharkdp/bat

  2. ripgrep (rg)

    用 Rust 重写的 grep，速度比 grep / ack / silver_searcher 都快好几倍。默认尊重 .gitignore，递归搜索整个项目秒出结果。VS Code 内部搜索就是用的它。

    https://github.com/BurntSushi/ripgrep

  3. fd

    find 的友好版：默认忽略 .git/.gitignore、彩色输出、并行执行。语法直观（fd pattern 不用 -name），常用 80% 用例都比 find 短得多。

    https://github.com/sharkdp/fd

  4. eza (ls 替代)

    ls 的现代版（exa 的活跃 fork）：彩色 + 图标 + Git 状态 + 树状视图。`eza --tree --git` 一行看到项目结构 + 哪些文件被修改了。

    https://github.com/eza-community/eza

  5. fzf

    通用模糊搜索器，所有命令的"灵魂"。Ctrl+T 模糊找文件、Ctrl+R 模糊找历史命令、git checkout 用 fzf 选 branch……装上之后整个 shell 体验改写。

    https://github.com/junegunn/fzf

  6. zoxide (z) — 智能 cd

    cd 的进化版：记住你常去的目录，输入 `z proj` 就能跳到 ~/work/projects/some-project。比 autojump 快、跨平台、Rust 实现。

    https://github.com/ajeetdsouza/zoxide

  7. jq — JSON 命令行处理器

    CLI 必备：从 API 响应里提取字段、过滤、变形 JSON。`curl ... | jq .data[0].name` 这种黄金组合救命无数次。语法初学有点小怪，掌握后回不去。

    https://github.com/jqlang/jq

  8. yq — YAML 版 jq

    处理 YAML / JSON / XML 的命令行工具，语法和 jq 高度兼容。改 K8s manifest、CI 配置、Helm values 时秒杀手动编辑。

    https://github.com/mikefarah/yq

  9. xh — curl 但人类友好

    HTTPie 的 Rust 重写版，二进制单文件、启动快。`xh httpbin.org/get foo==bar` 比 curl 那一长串选项清爽 10 倍。开发调试 API 必装。

    https://github.com/ducaale/xh

  10. curlie — curl + httpie 混血

    保留 curl 完整选项的同时给你 httpie 风格的输出（自动着色、JSON 格式化）。从 curl 平滑过渡的最佳选择，老命令完全兼容。

    https://github.com/rs/curlie

  11. dust — du 的可视化版

    du -sh 看磁盘占用太丑了。dust 直接给你一棵树，柱状图标出每个目录占了多少。秒定位"哪个鬼东西占了 50G"。

    https://github.com/bootandy/dust

  12. duf — df 的彩色版

    df 输出又乱又丑，duf 给你一张漂亮的表格：本地磁盘 / 网络挂载 / 临时文件系统分组显示，使用率有色条。Go 实现，跨平台。

    https://github.com/muesli/duf

  13. hyperfine — 命令耗时基准测试

    想知道你的脚本两个版本哪个更快？`hyperfine "v1.sh" "v2.sh"` 自动跑多次取平均，给出统计学意义的结论。比手动 `time` 严谨 100 倍。

    https://github.com/sharkdp/hyperfine

  14. tldr (tealdeer)

    man 页太啰嗦？`tldr tar` 直接给你 5 行最常用的 tar 命令。tealdeer 是 Rust 实现的快速版，启动毫秒级。

    https://github.com/dbrgn/tealdeer

  15. glow — 终端里读 Markdown

    README.md 在 cat 里看一片乱码？glow 渲染成漂亮的彩色版本，支持滚动、目录、表格、code block。本地或在线（GitHub）的 markdown 都能读。

    https://github.com/charmbracelet/glow

  16. watchexec — 文件变化触发命令

    改文件 → 自动跑测试 / 重启服务 / 重新生成。`watchexec -e py pytest` 监听 .py 文件改动自动跑 pytest。Rust 实现，比 nodemon 快还跨语言。

    https://github.com/watchexec/watchexec

  17. lazygit

    Git 命令行用户的福音：全屏 TUI 界面，鼠标 / 键盘都能操作。stage / commit / branch / log / rebase / cherry-pick 全在一屏。装上一周后回不去 git CLI。

    https://github.com/jesseduffield/lazygit

  18. lazydocker

    Docker 版的 lazygit。看容器列表、查日志、进 shell、删镜像、清缓存全在 TUI 里。比 docker ps + docker logs 来回切方便 10 倍。

    https://github.com/jesseduffield/lazydocker

  19. k9s

    Kubernetes 集群的 TUI 神器。pod / service / configmap / 日志 / 进 shell / 删 / 重启 全键盘搞定。装一次，告别 kubectl 百次切换。

    https://github.com/derailed/k9s

  20. btop — 美到爆的系统监控

    top / htop 的颜值天花板。CPU / 内存 / 磁盘 / 网络全图表化，进程树鼠标点击操作。装上之后你的服务器/Mac 监控会被同事盯上问"这是啥"。

    https://github.com/aristocratos/btop

  21. helix — 模式编辑器现代版

    不是 vim 的另一个 vim：开箱即用的模态编辑器，Tree-sitter 语法高亮、内置 LSP、多光标。零配置就能写代码，配置文件比 nvim 简单 10 倍。

    https://github.com/helix-editor/helix

  22. Neovim

    Vim 的现代复活版：原生 Lua 配置、内置 LSP、Tree-sitter、异步插件。配合 LazyVim / NvChad / AstroNvim 这种发行版，10 分钟就能搭出 IDE。

    https://github.com/neovim/neovim

  23. atuin — 加强版 Shell 历史

    替代 Ctrl+R 的本地历史搜索，加上"按目录 / 按时间 / 按上次退出码"过滤。可选云端同步多机历史（端到端加密）。再也不会"上次那条命令长啥样"。

    https://github.com/atuinsh/atuin

  24. tig — Git 日志查看器

    比 git log 漂亮 100 倍的 TUI 浏览器。回车看 commit diff、空格 stage hunks、c 创建 commit。比 lazygit 更轻量，老 vim 用户的习惯延伸。

    https://github.com/jonas/tig

  25. Starship — 跨 shell 的极速 prompt

    一个 prompt 配置走遍 zsh / bash / fish / pwsh。Rust 实现毫秒级渲染，自动显示 git 分支 / Node 版本 / Python venv / k8s 上下文……装上之后所有 shell 一致体验。

    https://starship.rs/

  26. Oh My Zsh

    zsh 的"开箱即用"配置框架。300+ 主题、200+ 插件（git / docker / kubectl 自动补全等），一行命令安装。新机器 zsh 起步标配。

    https://ohmyz.sh/

  27. fish shell

    "开箱即用"的现代 shell：自动补全、语法高亮、跨会话历史无需任何配置。和 bash / zsh 不完全兼容（脚本要稍改），但日常交互体验最佳。

    https://fishshell.com/

  28. Zellij — 现代终端复用器

    tmux 的现代版：开箱即用、UI 友好（状态栏 + 快捷键提示）、可视化窗口分屏。配置用 KDL / YAML 不用看 tmux 那种密码本。Rust 写的。

    https://zellij.dev/

  29. tmux — 经典终端复用器

    tmux 是 Unix 服务器管理者的肌肉记忆。SSH 进服务器跑长任务、断开重连不丢、多窗口切来切去——服务器场景下还是它最稳。Zellij 现代但 tmux 还是无处不在。

    https://github.com/tmux/tmux

  30. aider — 终端 AI 结对程序员

    用大模型修改代码的命令行工具，自动 git commit、可指定多文件 context、支持 Claude/GPT/Gemini/本地 Ollama。比 Cursor / Copilot 更"agent"，让 AI 直接动手而不是建议。

    https://aider.chat/

  31. Shell GPT (sgpt)

    在终端里用 ChatGPT。`sgpt "把所有 .log 文件按时间排序"` → 直接给你命令并可一键执行。结合 shell 历史用尤其顺手。支持 Claude / 本地模型。

    https://github.com/TheR1D/shell_gpt

  32. gping — 带图表的 ping

    ping 但实时画延迟图表。一眼看到网络抖动 / 丢包模式，比一行行数字直观 10 倍。Rust 实现。

    https://github.com/orf/gping

  33. dog — dig 的现代版

    dig 的彩色化更人性化版本：默认显示彩色、JSON 输出可选、语法更短（`dog example.com MX` 直接到位）。Rust 写的。

    https://github.com/ogham/dog
